工作笔记

工作中会用到的一些terminology

  • RTB (real time bidding) 实时竞价
  • DSP (demand side platform ）需求方平台，面向广告主的广告投放平台
  • DMP (Data Management Platform)
  • SSP (Sell-side platform) 媒体服务平台
  • CPC (Cost per click) 每次点击付费广告，当用户点击某个网站的CPC之后，站长酒就会获得相应收入
  • CPM (Cost Per 1000 impressions) 千人成本：由某一媒介或媒介广告排期表所送达1000人所需的成本. CPM是按曝光次数收费，适用于印象传播：认用户知道有这么一件事/商品 (CPM is mainly used when you want to increase your brand /product recognition.You could use certain tools to make an AD appear a number of times to a single user-it's all about creating the impression. )
  • CPA (Cost per action) 按照广告投放的实际效果
  • PMP(Private Marketplace)
  • CTR (Click through rate ) 点击率 点进率 = 广告被点击次数 / 广告被显示次数
  • 对关系的一个简单说明: 比如说你打开一个网页, 就说是新浪好了, 上面挂了一个RTB的广告位, 那么当页面加载这个广告位的时候, 新浪就会向Exchange平台(就说是百度好了)发起一个消息, 说"我这儿有一个人要看广告了, 你给我发一个广告过来吧", 百度收到这个消息之后, 他会再向每一个接入到他的平台的要买广告的人(一般来说是DSP)发消息, 说"这儿有一个新浪的广告位要出广告, 你们看愿意出多少钱买这次曝光", 然后DSP那边一通计算, 告诉了百度一个数, 然后百度从每个DSP出的价格中挑选出最合适的那个(不求甚解的话就理解成出价最高的那个就好了, 实际情况中一般是第二高的价格), 把这次广告的曝光机会给这个DSP, 然后把这个DSP提供的广告返回到新浪那边, 然后新浪最终把这个广告展示出来, 你看到了这个广告. 这一系列过程都发生在你等待网页加载的1秒左右的时间里, 每个环节都只有几十到一百毫秒的处理时间,所以叫实时竞价. 而且在这个过程中, 媒体方或平台方可以告诉DSP这边关于这个广告位以及这个用户的信息, 比如这个用户是男是女, 年龄, 地区等等, DSP可以根据这些信息做更合理的出价, 并找到更符合广告主需求的目标人群.此外自然还能很方便的监测曝光率,点击率之类的指标.

1.理解AngularJS组件

  • Modules:
    • The module of an app is where we'll contain all of our application code. An app can contain several modules, each one containing code that pertains to specific functionality.

  • Controllers:
    • The controller in AngularJS is a function that adds additional functionality to the scope of the view. We use it to set up an initial state and to add custom behavior to the scope object.

  • Directives:
    • Directives are Anugular's method of creating new HTML elements that have their own functionality.

  • Dependency Injection:
    • ?

  • Services:
    • In angularJS, every time we switch a route or reload a view, the current controller gets cleaned up by Angular. Service provide a method for us to keep data around for the lifetime of the app and communicate across controllers in a consistent manner.

  • Promises:
    • A promise is a method of resolving a value in an asynchronous manner. Promises are objects that represent the return value or thrown exception that a function may eventually provide. 代替callback的
    The real point of promises is to make asynchronous functions look more like synchronous ones. With synchronous functions, we can capture both return values and exception valued as expected.

  • Events:
    • Just as the browser respond to browser-level events, such as a mouse click or a page scroll, our Angular app can respond to Angular events. This fact gives us the advantage of being able to communicate across our application inside nested components that are not built with other components in mind.
    We can think of events as sinppets of information propagated across an appilcation that generally contain information about what's happening inside of that application.
